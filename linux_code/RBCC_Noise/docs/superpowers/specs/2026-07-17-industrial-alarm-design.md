# 厂房噪声感知型自适应警告器设计

## 1. 项目目标

构建一个兼容 Windows 笔记本与树莓派的 Python 原型系统。系统能够从实时麦克风或 WAV 文件获取厂房噪声，使用 STFT 和规则式特征识别分析噪声频谱，输出三个最佳警告频率，自动合成并播放“短—短—长”多频警告音，同时保存频谱图、CSV、JSON 和 WAV 结果。

## 2. 范围与约束

- 同时提供命令行与 Tkinter 图形界面。
- 输入支持实时麦克风与 WAV 文件。
- 核心算法不依赖训练数据，不使用 SVM、强化学习或深度学习。
- 默认分析采样率 16 kHz，候选警告频率范围 800–4000 Hz。
- 默认输出三个频率，任意两个频率至少间隔 350 Hz。
- 警告音采用固定“短—短—长”节奏；频率根据噪声频谱自适应变化。
- 系统设置数字限幅，避免合成音削波。
- 实时分析采用后台线程，GUI 关闭或停止时必须释放音频流。
- 在无音频设备的开发或测试环境中，算法测试仍可运行。

## 3. 架构

项目分为配置、音频输入、噪声分析、频率选择、警告音生成、结果导出、命令行和图形界面八个模块。核心模块只接收 NumPy 数组和配置对象，不直接依赖 GUI，从而保证 Windows 与树莓派共用同一套算法。

```text
麦克风 / WAV 文件
        ↓
音频标准化与分帧
        ↓
STFT、短时能量、频谱平坦度、峰值检测
        ↓
规则式噪声类型识别
        ↓
候选频率局部噪声能量与峰值距离评分
        ↓
选择三个相互分离的最佳警告频率
        ↓
合成“短—短—长”多频警告音并限幅
        ↓
播放与导出频谱图、CSV、JSON、WAV
```

## 4. 模块职责

### `config.py`
定义采样率、分析时长、STFT 参数、候选频率范围、频率间隔、报警节奏和输出目录等配置。

### `audio_input.py`
枚举输入设备；从默认或指定麦克风采集固定时长音频；读取 WAV 并转换为单声道浮点数组；为实时模式提供可停止的块式音频流。

### `noise_analyzer.py`
计算 STFT 功率谱、平均频谱、短时能量、频谱平坦度、频谱峰值和噪声类型。噪声类型包括稳态背景、固定频率、瞬态冲击、宽频混合和低电平输入。

### `frequency_selector.py`
在 800–4000 Hz 范围内建立候选频率。评分综合局部噪声能量、与主要噪声峰值的距离、听觉范围偏好和上一次结果稳定性；使用贪心选择保证频率间隔。

### `alarm_generator.py`
根据三个频率合成带淡入淡出的“短—短—长”多频警告音；完成归一化、限幅、WAV 保存和可选播放。

### `result_exporter.py`
创建带时间戳的输出目录，保存频谱图、噪声频谱 CSV、分析 JSON 和警告 WAV。

### `main_cli.py`
提供麦克风和文件两种模式，打印噪声类型、主要噪声峰值、最佳频率和处理耗时，可选择播放与导出。

### `main_gui.py`
提供输入模式、设备、文件、分析时长、音量、开始、停止、播放和保存控件；嵌入频谱图，后台执行采集与分析，主线程更新界面。

## 5. 频率评分

对候选频率 \(f\) 计算：

\[
Score(f)=0.55E(f)+0.25P(f)+0.10H(f)+0.10S(f)
\]

- \(E(f)\)：候选频率附近噪声越低，得分越高。
- \(P(f)\)：距离已检测噪声峰值越远，得分越高。
- \(H(f)\)：对 1200–3500 Hz 的中频区域给予温和偏好。
- \(S(f)\)：与上一轮选择接近时给予稳定性奖励。

选择时按分数从高到低遍历，已选频率 ±350 Hz 范围内的候选频率不再选择。若不足三个频率，则逐步放宽最小间隔，但不得低于 200 Hz。

## 6. 噪声分类规则

- 低电平输入：RMS 低于阈值。
- 瞬态冲击：短时能量的变异系数和峰均比较高。
- 固定频率：频谱中存在突出窄峰，且频谱平坦度较低。
- 宽频混合：频谱平坦度较高，多个频带同时有明显能量。
- 稳态背景：不满足以上条件且短时能量较稳定。

分类用于解释和调整策略，不作为最佳频率选择的唯一依据。

## 7. 输出

每次导出创建：

```text
output/YYYY-MM-DD_HHMMSS/
├── spectrum.png
├── noise_profile.csv
├── analysis_result.json
└── generated_alarm.wav
```

JSON 包括配置、噪声类型、RMS、频谱平坦度、主要峰值、最佳频率、各频率评分和处理耗时。

## 8. 错误处理

- 未安装或无法加载 `sounddevice`：文件分析仍可运行，并给出清晰提示。
- 没有输入/输出设备或设备被占用：捕获异常并提示设备信息。
- WAV 为空、格式损坏或采样率异常：拒绝分析并说明原因。
- 输入削波：在结果中标记 `input_clipping=true`。
- 输入过低：输出保守的默认频率并标记低电平。
- 实时线程停止：使用 `threading.Event`，始终在 `finally` 中关闭音频流。

## 9. 测试标准

- 输入 1000 Hz 强窄带噪声时，不选择 1000 Hz 附近频率。
- 输入多个窄带峰值时，输出避开所有主要峰值。
- 三个输出频率满足最小间隔要求。
- 同一噪声重复分析时结果稳定。
- 合成警告音峰值不超过 1.0，不含 NaN 或 Inf。
- WAV 输入完成正确的单声道转换与重采样。
- 结果导出生成四种文件并包含必要字段。
- 无音频硬件时，单元测试全部通过。
